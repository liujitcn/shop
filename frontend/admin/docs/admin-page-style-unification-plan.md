# 后台页面风格统一修改方案

## 1. 背景与目标

当前项目中，以下页面与后台其他业务页在视觉风格上存在明显差异：

- 工作台
- 数据分析
- 商品详情
- 商品添加
- 商品编辑
- 订单详情
- 订单发货
- 发货详情
- 退款
- 退款详情

这些页面当前更偏向“专题页 / 驾驶舱”风格，存在大面积渐变背景、超大圆角、强装饰 Hero 区、展示型指标卡等表现方式。与项目内大部分采用 `table-box + el-card + ProTable / ProForm / ProDialog` 的后台页面相比，页面观感割裂较强，打开后容易显得突兀。

本次调整目标如下：

- 将上述页面的视觉语言收敛到项目当前后台主风格。
- 保留现有业务流程、接口调用、数据结构和交互路径。
- 优先做样式与页面结构层面的统一，不额外引入复杂抽象。
- 让这些页面在打开时“不突兀”，并和现有列表页、弹窗页形成一致体验。

## 2. 当前问题归纳

### 2.1 视觉风格偏离

- 页面级渐变背景和放射光斑较重，与常规后台浅灰底风格不一致。
- 大量使用 24px 级别大圆角和较强阴影，卡片显得过于“展示化”。
- 顶部 Hero 区占首屏较多空间，信息密度偏低。
- 指标卡颜色和装饰感偏强，更像数据看板或活动页，而不是后台业务页。

### 2.2 页面结构不统一

- 有的页面使用“头图区 + 指标区 + 内容区”的专题页结构。
- 有的页面是详情页，但仍沿用展示型首屏，不符合后台信息查看习惯。
- 发货 / 退款相关弹窗内部结构较重，与当前列表页弹窗风格不一致。

### 2.3 体验层面的割裂

- 从列表页进入详情、编辑、弹窗时，视觉跳变明显。
- 同一业务链路中，列表页是标准后台风格，详情和弹窗却切换为另一套视觉体系。
- 多个目标页面之间虽然都偏“重设计”，但彼此也没有完全统一的组件和层级规则。

## 3. 统一设计原则

### 3.1 页面整体风格

- 页面背景统一回归项目常规浅灰底，不再使用大面积页面级渐变背景。
- 页面容器优先贴合现有后台内容区留白与宽度习惯。
- 保持简洁、稳定、低干扰的信息呈现方式。

### 3.2 卡片风格

- 卡片圆角统一收敛到 `12px - 16px` 区间。
- 阴影减轻，边框与底色更接近当前常规 `el-card` 观感。
- 标题、说明、统计块等采用统一的间距、字号和层级规则。

### 3.3 信息层级

- 详情页优先采用“基础信息卡 + 分组信息卡 + 表格卡”结构。
- 编辑页优先采用“页头说明 + 步骤条 + 表单区”结构。
- 弹窗页优先采用“顶部摘要 + 信息卡 + 表单 / 表格区”结构。
- 不再使用大块 Hero 区承载介绍性文案。

### 3.4 组件使用原则

- 优先复用现有 `el-card`、`ProTable`、`ProForm`、`ProDialog`。
- 只抽离高复用、低业务耦合的轻量样式，不下沉业务流程。
- 不为了统一风格引入新的重型页面组件。

## 4. 页面级修改方案

## 4.1 工作台

对应页面：

- `src/views/dashboard/workspace/index.vue`

调整方向：

- 去掉当前首页式 Hero 区和大面积装饰背景。
- 保留欢迎信息，但收敛为普通页头或首张信息卡。
- 将“今日关注”“工作建议”“快捷入口”统一组织为常规业务卡片。
- 将概览卡片调整为简洁统计项，降低展示感和装饰感。

目标效果：

- 更像后台首页工作面板，而不是运营门户首页。
- 保留重点入口和待办信息，但首屏更紧凑、更稳。

## 4.2 数据分析

对应页面：

- `src/views/dashboard/analytics/index.vue`
- `src/views/dashboard/analytics/components/OrderBarChart.vue`
- `src/views/dashboard/analytics/components/SalesBarChart.vue`
- `src/views/dashboard/analytics/components/GoodsPieChart.vue`
- `src/views/dashboard/analytics/components/OrderPieChart.vue`

调整方向：

- 保留时间切换和汇总数据，但去掉 Hero 主卡式布局。
- 顶部四个统计指标改为标准统计卡样式。
- 图表组件统一为标准卡片容器，收敛边框、阴影、标题区样式。
- 降低装饰色和渐变使用比例，强调业务信息本身。

目标效果：

- 更贴近后台“数据查看页”，而不是独立驾驶舱页面。
- 与普通页面切换时视觉跳变更小。

## 4.3 商品详情

对应页面：

- `src/views/goods/info/detail/index.vue`

调整方向：

- 去掉当前大图 Hero 首屏结构。
- 调整为“基础信息卡 + 图片信息卡 + SKU / 规格 / 属性标签页”。
- 商品主图缩小到信息区内展示，不再占据整块首屏主视觉。
- 轮播图数量、详情图数量、SKU 数量等概览保留，但改为简洁统计项。
- 标签页和卡片外观统一回归后台业务风格。

目标效果：

- 保持信息完整，同时让页面更像商品后台详情，而不是商品展示页。

## 4.4 商品添加 / 商品编辑

对应页面：

- `src/views/goods/info/edit/index.vue`
- `src/views/goods/info/edit/components/info.vue`
- `src/views/goods/info/edit/components/prop.vue`
- `src/views/goods/info/edit/components/sku.vue`

调整方向：

- 页面头部从“商品编辑工作台”改为标准页头说明。
- 步骤条保留，但整体卡片和布局更贴合后台表单页。
- 收敛顶部指标卡的样式和体量。
- 同步调整三个子表单组件的卡片表现，避免子模块仍保留强装饰视觉。
- 新增与编辑场景共用同一套稳态页面框架。

目标效果：

- 打开新增或编辑页时，视觉感知更接近标准后台多步骤表单页。
- 从商品列表进入编辑页时观感连续。

## 4.5 订单详情

对应页面：

- `src/views/order/info/detail/index.vue`

调整方向：

- 顶部金额概览保留，但改为轻量统计卡。
- 基础信息、地址、商品清单、支付信息、物流信息、退款信息统一为同一套 Section Card 样式。
- 物流时间线保留，但弱化外层装饰。
- 页面整体背景、边框、圆角、阴影统一收敛。

目标效果：

- 页面更像标准后台订单详情页。
- 与订单列表页、发货弹窗、退款弹窗形成一致的视觉语言。

## 4.6 订单发货 / 发货详情 / 退款 / 退款详情

对应页面：

- `src/views/order/info/index.vue`

说明：

- 这几项当前不是独立路由页面，而是订单列表页中的弹窗流程。

调整方向：

- 发货弹窗与退款弹窗统一采用“顶部摘要 + 信息卡 + 表单卡 / 表格卡”结构。
- 去掉弹窗中的 Hero 卡和高装饰指标区。
- 保留必要的订单、物流、支付、退款摘要信息。
- 发货详情和退款详情与订单详情页统一 Section Card 规范。
- 表单区域继续使用 `ProForm`，但外观收敛为普通业务表单卡。

目标效果：

- 从订单列表打开弹窗时，依然保持后台页的一致性。
- 不再出现“列表很稳，弹窗像另一套产品”的体验割裂。

## 5. 建议的实现方式

### 5.1 第一阶段

- 仅调整模板结构和页面样式。
- 不修改接口请求逻辑。
- 不修改现有业务判断和权限控制。
- 不修改路由结构。

### 5.2 第二阶段

- 提炼重复的页面级样式。
- 如有必要，可沉淀轻量公共样式片段，例如：
  - 统一详情页容器
  - 统一 Section Header
  - 统一 Summary Item
  - 统一弹窗信息卡样式

### 5.3 第三阶段

- 对数据分析图表组件、商品编辑子组件等局部样式做统一收口。
- 避免父页面风格收敛后，内部子组件仍保留明显旧视觉。

## 7. 实施步骤

1. 统一工作台、数据分析页的页面基础框架和卡片风格。
2. 统一商品详情、商品新增、商品编辑相关页面的头部与内容卡片风格。
3. 统一订单详情页和订单发货 / 退款弹窗的视觉结构。
4. 抽取可复用的轻量公共样式，减少重复实现。
5. 更新 `README.md`，同步说明风格约定与相关实现方式。
6. 对本次涉及的代码执行 ESLint 校验并处理报错。

## 8. 风险与边界

- 本次默认只处理风格统一，不做业务流程重构。
- 不将订单发货 / 退款弹窗改成独立页面。
- 不为了追求抽象而改造基础组件接口。
- 如果后续确认需要“更朴素”或“保留少量质感”，可在正式实施前再微调视觉基调。

## 9. 预期结果

完成后，工作台、分析、商品详情 / 编辑、订单详情、发货 / 退款相关弹窗将回归同一套后台设计语言：

- 页面不再突兀
- 首屏更紧凑
- 信息层级更清晰
- 与现有列表页和弹窗页的切换更自然
- 后续新增类似页面时也更容易延续统一风格
