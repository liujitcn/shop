# Codex 规则（仓库根）

## 仓库结构
- `backend`：Go + Kratos 后端；后端业务、接口契约、数据库访问、服务启动改动统一在此完成。
- `frontend/admin`：管理后台；页面、组件、样式、构建配置改动统一在此完成。
- `frontend/app`：uni-app 商城端；页面、分包、商城端接口调用与多端兼容改动统一在此完成。
- `sql`：初始化数据、权限数据与演示数据脚本。
- 模块业务代码不要散落到其他模块目录，避免跨目录职责漂移。

## 跨模块协作
- 后端接口、前端调用、SQL 初始化数据存在依赖时必须一致改动，按“接口契约 → 后端实现 → 前端调用 → SQL/文档”顺序梳理影响范围，避免只改其中一层。
- 生成代码、协议产物、依赖注入、自动导入文件统一用各模块既有命令生成，不手写伪生成结果。
- 生成命令执行成功后，其产物（`api/gen`、`pkg/gen`、`rpc` 等目录下文件）无需人工逐行验证正确性，以生成命令是否报错为准，避免重复读取全量生成文件。
- 涉及推荐、认证、上传、静态资源挂载、后台与商城共用能力时，先确认是否影响 admin 与 app 两个前端入口。

## 文档
- 调整顶层目录结构、启动方式、公共命令或模块关系时，必须同步更新根 `README.md`。
- 仓库级规则与文档保持轻量，只描述跨模块通用约束，不重复抄写子模块细则。
